29 /***********************************************************************
30 * The structure FHV describes sparse updatable FHV-factorization.
31 *
32 * The FHV-factorization has the following format:
33 *
34 * A = F * H * V, (1)
35 *
36 * F = P0 * L * P0', (2)
37 *
38 * H = H[1] * H[2] * ... * H[nfs], (3)
39 *
40 * V = P * U * Q, (4)
41 *
42 * where: A is a given (unsymmetric) square matrix; F, H, V are matrix
43 * factors actually computed; L is a lower triangular matrix with unity
44 * diagonal; U is an upper tringular matrix; H[k], k = 1, 2, ..., nfs,
45 * is a row-like factor, which differs from unity matrix only in one
46 * row called a non-trivial row; P0, P, Q are permutation matrices; and
47 * P0' is a matrix transposed to P0.
48 *
49 * Matrices F, V, P, Q are stored in the underlying LUF object.
50 *
51 * Non-trivial rows of factors H[k] are stored as sparse vectors in the
52 * right (static) part of the sparse vector area (SVA). Note that unity
53 * diagonal elements of non-trivial rows are not stored.
54 *
55 * Matrix P0 is stored in the same way as matrix P.
56 *
57 * Matrices L and U are completely defined by matrices F, V, P, and Q,
58 * and therefore not stored explicitly. */
59
